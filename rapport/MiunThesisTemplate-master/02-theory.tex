\subsection{Kunskapsgraf}\label{subsec:Kunskapsgraf}
En kunskapsgraf är ett designmönster som används för att organisera och representera data där fokus ligger på relationer mellan olika entiteter.
Istället för att lagra information i tabeller eller dokument, struktureras data i form av noder (entiteter) och relationer som kopplar samman dessa.
Detta möjliggör analys och förståelse av komplexa samband mellan noder\cite{5}.

Kunskapsgrafer är särskilt användbara i situationer där data är starkt sammankopplad och där relationerna i sig bär viktig information. De används exempelvis inom rekommendationssystem, bedrägeridetektion och avancerad informationssökning.
Genom att integrera data från olika källor och organisera den utifrån tydliga samband kan kunskapsgrafer bidra till att synliggöra mönster och insikter som annars kan vara svåra att upptäcka\cite{5}.

\subsubsection{Grafdatabasen Neo4j}

Grafdatabaser är en typ av databashanteringssystem som representerar data i from av noder och relationer.
Till skillnad från traditionella relationsdatabaser är relationerna en central del av datamodellen, vilket bidrar till effektiv traversering av data(s.~6)\cite{robinson2015graph}. 

Grafdatabaser baseras på riktade grafer, vilket innebär att varje relation har en startnod och därmed en definierad riktning.
Denna riktning används för att beskriva semantiken i relationen mellan två noder.
Trots detta kan relationer traverseras i båda riktningarna vid frågeställning\cite{10}.

Det är inte heller nödvändigt att alla noder i en graf är direkt kopplade till varandra, utan en graf kan bestå av flera sammanhängande delgrafer.

Neo4j är grafdatabasen som används i detta projekt för att implementera kunskapsgrafen. I Neo4j är grafstrukturen implementerad direkt i databasen på lagringsnivå, och inte ovanpå an annan databasmodell.
Detta innebär att data lagras direkt i grafstrukturen, vilket bidrar till effektiv hantering av relationer och traversering av data\cite{6}.


\subsubsection{Cypher}
Cypher är det frågespråk som används i Neo4j. Det är ett deklarativt frågespårk för grafdatabaser ursprungligen utvecklat för just Neo4j.
Språket är utformat för att uttrycka frågor genom mönstermatchning, vilket gör det möjligt att formulera graftraverseringar på ett naturligt sätt \cite{11}.

Syftet med cypher somfrågespråk är att förenkla frågeställningar i grafdatabaser genom att möjliggöra analy av relationer utan join-operationer. Uttrycka graftraversering på ett naturligt sätt. Hantera skapande, uppdateringar och borttagning av data i grafen\cite{11}.

En cypher fråga ser ut på följande sätt:
\begin{verbatim}
MATCH (p:Person)-[:WORKS_ON]->(r:Repository)
RETURN r
\end{verbatim}
Denna fråga hämtar alla noder av typen Repository som är kopplade till noder av typen Person genom relationen WORKS\_ON.

Som vi ser i exemplet ovan är språket väldigt intuitivt och lätt att förstå.


\subsection{Large Language Model (LLM)}\label{subsec:LLM}
En LLM är är en typ av artificiell intelligensmodell som är tränad på stora mängder data. Detta gör dem kapabla till att förstå det mänskliga språket samt andra typer av innehåll för att kunna utföra olika typer av arbetsuppgifter\cite{12}.

I detta projekt kommer LLM'en att användas för att tolka en användares fråga och sedan tolka vilket eller vilka verktyg som ska användas för att returnera den information som efterfrågats.

\subsection{Model Context Protocol (MCP)}\label{subsec:mcp_theory}
MCP (Model Context Protocol) är ett protokoll som används för att låta applikationer förse LLM'er med kontextuell information.
MCP gör det möjligt för olika AI-modeller att ansluta till olika datakällor och verktyg. Detta gör det enklare att utveckla AI-applikationer utan att behöva skapa specialiserade gränssnitt för varje datakälla eller modell\cite{4}.

MCP består av tre huvudroller: servrar, klienter och värdar. Servrarna är programmen som tillhandahåller verktyg och dataåtkomstmöjligheterna för LLM'er att använda.
Varje MCP server tillhandahåller en uppsättning verktyg, vilka är de som ansvarar för att hämta informationen från datakällorna. Klienterna är de som sammankopplar LLM'en med MCP servern. De ansvarar för att ta emot förfrågningar från LLM'en, vidarebefordra dessa till motsvarande MCP server och returnera resultatet från servern tillbaka till LLM'en.
Värdarna är de som vill få åtkomst till data genom MCP. Exempelvis program som tillhandahåller ett användargränssnitt för att interagera med LLM'er, medan de själva integrerar MCP klienten för att kunna ansluta MCP servrarna och därmed utöka LLM'ens funktion med de verktyg som tillhandahålls av servrarna\cite{4}.

\subsection{Cherry Studio}\label{subsec:CherryStudio} 
Cherry Studio är en integrerad AI-plattform som kombinerar flertalet funktioner såsom multimodellstöd, kunskapsbashantering, översättning och AI-genererad bildskapning\cite{7}.

Systemet erbjuder också användandet av MCP servrar för att ansluta till externa datakällor och verktyg, vilket gör det möjlgit att utöka funktionaliteten i LLM'er som används inom plattformen.
Det går att använda befintliga MCP servrar eller att skapa sina egna för att ansluta till specifika datakällor eller verktyg som inte redan är integrerade i Cherry Studio.
